🛠️ The Implementation Strategy: Static Localization Dictionary
Since your app text is fixed (titles, quiz scenarios, risk alerts), the fastest and most efficient way to do this in Streamlit is using a Localization Dictionary. You don't need heavy external translation libraries like Google Translate API (which require paid API keys and cause network delays).

Step 1: Create a Language Selector Sidebar
Add a simple dropdown at the top of your app.py script so users can toggle their preferred language:

Python
# Place this at the very top of app.py, right after page configuration
language = st.sidebar.selectbox("🌐 Choose Language / भाषा चुनें", ["English", "Hindi (हिंदी)"])
Step 2: Define the Translation Dictionary
Create a structured dictionary holding your UI strings for both English and Hindi. Paste this right below your language selector:

Python

Step 3: Replace Hardcoded Strings with Dictionary Keys
Now, update your UI elements to use the lang_set keys instead of raw text. For example:

Old Code: st.title("🛡️ UPIShield: UPI Fraud Awareness Portal")

New Code: st.title(lang_set["title"])

Old Code: amount = st.number_input("Transaction Amount (in ₹):", ...)

New Code: amount = st.number_input(lang_set["amount"], ...)

Old Code: if st.button("Scan Transaction Risk"):

New Code: if st.button(lang_set["scan_btn"]):
